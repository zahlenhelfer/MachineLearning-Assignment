Practical Machine Learning

Abstract

With this assignment a machine for learning algorithm to predict activity quality is build. I uses data from activity monitors. The used data for this assignment can be found online here [1].

The Data Retrieval

The dataset from [2] and [3] can be downloaded with these handy calls. It first checks if the file exists locally, if not it gets downloaded:

if (! file.exists('./training-data.csv')) {
  download.file('http://d396qusza40orc.cloudfront.net/predmachlearn/pml-training.csv',   destfile = './training-data.csv')
}
if (! file.exists('./testing-data.csv')) {
  download.file('http://d396qusza40orc.cloudfront.net/predmachlearn/pml-testing.csv',   destfile = './testing-data.csv')
}

The data is formated in CSV. Which can be easly loaded into R using read.csv:

mldata.training <- read.csv('./training-data.csv')
mldata.testing  <- read.csv('./testing-data.csv')

Predictive Model

The forest algorithm 4 is been chosen for the building of a predictive model. It is, because it works good with non-linear models. Its not needed to select parameter for this algorithm and run robust to correlated covariates and outliers. Futher more, the hign number of dimensions and observations makes random forest a good choice. For more details look at 4.

library(randomForest)

The random forest classifier isnt deterministic. For the matter of reproduction it is set to a random value (birth date of my child here).

set.seed(310513)

The next step is to train the classifier. This is done by using all of the independent variables and 2048 trees.

myModel <- randomForest(classe ~ ., data = proc.mldata.training, ntree = 2048)
